01

¿Información o Dato?

Dato

Hecho, evento o transacción registrado sin procesar. Es la entrada bruta a partir de la cual se produce la información. Por sí solo no tiene significado alguno.

→ "5%"

Información

Datos procesados y comunicados de tal manera que pueden ser entendidos e interpretados por el receptor. Sirven para tomar decisiones.

→ "Hemos incrementado un 5% las ventas en el último mes"

Los datos se convierten en información cuando se les añade significado mediante:

Contextualizando → Categorizando → Calculando → Condensando

La jerarquía del conocimiento:

DATOS → INFORMACIÓN → CONOCIMIENTO → SABIDURÍA
02

El Ciclo de Vida de los Datos

01

Captura

Recopilación por entrada manual, sensores, extracción de bases de datos.

02

Almacenamiento

Custodia en repositorios, archivos, bases de datos o la nube.

03

Procesamiento y visualización

Cribado, análisis estadístico, dashboards, gráficos, infografías.

04

Uso operativo

Toma de decisiones, optimización de procesos, mejora de eficiencia.

05

Almacen. / Eliminación / Reutilización

Archivar, eliminar según normativa de privacidad o reutilizar para nuevas iniciativas.

El ciclo de vida de los datos es el conjunto de etapas que los datos atraviesan desde su creación hasta su eliminación o archivado. Cuidar cada fase es clave para evitar problemas de seguridad, pérdida o desorden.

03

El Big Data

Big data: datos tan grandes, rápidos o complejos que es difícil o imposible procesarlos con los métodos tradicionales. Los analistas de big data recogen, almacenan y analizan datos para dotarlos de sentido y coherencia.

Desventajas del Big Data

📊 Calidad y cantidad

El enorme volumen hace difícil el análisis y provoca obsolescencia del software. No todos los datos son útiles; hay que depurarlos.

👤 Falta de personal

Es una ciencia en desarrollo con escasez de profesionales capacitados para manejar estas tecnologías.

🔐 Ciberseguridad

Las empresas son blanco de ciberdelincuentes. Los gobiernos deben legislar para proteger los datos.

💰 Costes elevados

Solo las grandes empresas pueden mantener la inversión necesaria en hardware, software y personal.

CASO REAL · UPS

UPS usa big data para organizar sus rutas de entrega, logrando un ahorro considerable de combustible y una reducción del tiempo de entrega. El cambio fue costoso: requirió inversión en hardware, software y personal especializado.

Las 5 V del Big Data

📦

VOLUMEN

La cantidad de datos generados sigue en constante aumento.

⚡

VELOCIDAD

El análisis exige rapidez. Mejor pocos datos en tiempo real que mucha info con lentitud.

🎛️

VARIEDAD

Texto, audio, imagen, vídeo, hojas de cálculo, redes sociales…

✅

VERACIDAD

Los datos deben ser ciertos. Los que no lo sean deben ser filtrados.

💎

VALOR

Las conclusiones obtenidas deben ser útiles para la toma de decisiones.

ℹ️ Ya hay quienes hablan de «las 10 V del big data»: añaden Visualización, Vulnerabilidad, Validación, Volatilidad y Variabilidad.

🤖 Inteligencia Artificial

Posibilidad de que las máquinas realicen tareas que requieren inteligencia humana. El ML y el DL posibilitan el aprendizaje automático y profundo de los sistemas IA.

⚙️ Machine Learning

Algoritmos y modelos estadísticos para que las máquinas aprendan a realizar tareas específicas sin intervención humana directa.

🧠 Deep Learning

Forma avanzada de ML que usa redes neuronales artificiales para tareas complejas: reconocimiento de imágenes, procesamiento del lenguaje natural.

EJEMPLO · COMERCIO ELECTRÓNICO

Una tienda online recopila big data de clics, compras y comentarios → aplica análisis de datos para identificar patrones → usa machine learning para personalizar recomendaciones → deep learning para reconocer productos en imágenes → la IA ajusta precios y ofrece chatbots automáticamente.

07

La Ciencia de Datos

Campo multidisciplinario que utiliza métodos científicos, procesos y sistemas para extraer conocimientos de datos. Incluye estadística, aprendizaje automático y visualización. Está presente en todo el ciclo de vida del dato.

01

Definición del problema

Detectar situaciones que deben mejorarse. Ej: en un bufete de abogados, la duración de los casos varía mucho → se quiere mejorar la asignación de recursos.

02

Adquisición y preparación de datos

Recoger datos relevantes para el interrogante planteado y prepararlos para el análisis. Puede implicar recopilar casos anteriores, documentos legales y datos de clientes.

03

Análisis exploratorio de datos (EDA)

Visualizar los datos para identificar tendencias y patrones. Ej: duración promedio de casos, frecuencia de tipos de litigios, relaciones entre variables clave.

04

Modelado de datos

Construcción de modelos predictivos o clasificatorios. Ej: modelo que predice la probabilidad de éxito en un caso basándose en características específicas.

05

Implementación de resultados y evaluación

Aplicar los resultados en la práctica y evaluar su efectividad. La empresa ajusta sus estrategias basándose en las predicciones del modelo.
